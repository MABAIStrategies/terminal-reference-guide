üñ•Ô∏è Understanding Different Terminals

What Terminal Am I Using?

üíô PowerShell (Windows)

Modern Windows terminal, uses $ for variables

PS C:\Users\You> whoami

Run: npm, python, docker, yaml

‚ö´ Git Bash (Windows)

Linux-like terminal for Windows, uses $ prefix

user@computer MINGW64 /c/Users

Run: git, bash, ls, mkdir, ./scripts

üçé Mac Terminal

Default Mac terminal (zsh), shows % prompt

username@MacBook-Pro ~ %

Run: git, bash, npm, python, ls

üêß Linux Terminal

Linux terminal, shows $ prompt (root shows #)

username@machine:~$ ls

Run: git, bash, npm, python, docker

‚úÖ How to Check Your Terminal

Windows: Look for "PowerShell" or "Git Bash" in the title bar. Or type Get-Host in PowerShell
Mac: Check prompt ends with % (zsh) or $ (bash). Type echo $SHELL
Linux: Usually clear. Type echo $SHELL to confirm bash vs zsh

üî¥ Windows-Specific Gotchas & Solutions

Common Windows Terminal Mistakes

‚ùå "command not found: ls"
‚úÖ Use Git Bash instead of PowerShell. PowerShell doesn't understand bash commands. Git Bash gives you bash commands on Windows.
‚ùå "command not found: npm"
‚úÖ Install Node.js. Restart your terminal after installation. Verify with: node --version
‚ùå "./script.sh is not recognized"
‚úÖ Use Git Bash, not PowerShell. Or run: bash script.sh
‚ùå "command not found: sudo"
‚úÖ Windows doesn't have sudo. Run PowerShell as Administrator instead (right-click ‚Üí Run as Administrator)
‚ùå "export VAR=value" doesn't work in PowerShell
‚úÖ PowerShell syntax: $env:VAR = "value". Or use Git Bash for bash syntax.
‚ùå "Cannot be loaded because running scripts is disabled"
‚úÖ PowerShell security policy. Run: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
